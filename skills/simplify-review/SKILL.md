# simplify-review

Provide findings-first review output.

## When To Use

- Code review.
- Plan review.
- Regression risk analysis.
- Architecture critique.
- Second-opinion validation.

## Output Rules

- Lead with findings-first output.
- Prioritize bugs, regressions, missing tests, and unclear assumptions.
- Keep the review focused on risk and correctness before summaries.

## Boundaries

- Offer peer-review style second opinions only when the relevant code or plan is available.
- Do not broaden into brainstorming unless the request is explicitly exploratory.
